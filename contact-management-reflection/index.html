通讯录管理系统总结 —— C 语言与数据管理的实践

作者: yituanxing
完成日期: 2025-04-06
分类专栏: C 语言学习笔记
标签: C 语言, 通讯录, 数据管理, 结构体, 排序
版权: 本文原创，转载请注明出处

专栏导航

本系列专栏 “C 语言学习笔记” 记录了我学习和实践 C 语言的过程，欢迎订阅以查看更多内容。

前言

我使用 C 语言实现了一个通讯录管理系统，支持添加、删除、查找、修改、显示和排序联系人信息。这个项目让我掌握了结构体的使用、数组操作以及 C 语言的输入输出处理。以下是我的学习与成长总结。

项目概述

通讯录管理系统是一个基于命令行的程序，用于管理联系人信息。系统支持以下功能： 1. 添加联系人：录入联系人信息（姓名、年龄、性别、电话、地址）。 2. 删除联系人：根据姓名删除指定联系人。 3. 查找联系人：根据姓名查找并显示联系人信息。 4. 修改联系人：根据姓名修改联系人信息。 5. 显示所有联系人：以表格形式展示所有联系人信息。 6. 排序联系人：支持按姓名或年龄排序。 7. 退出系统：安全退出程序。

代码分为头文件（contact.h）、实现文件（contact.c）和主文件（test.c）。核心数据结构是结构体数组 struct Contact，最大支持 100 个联系人。

学到的知识

1. C 语言核心特性

  • 结构体：使用 struct PeoInfo 存储联系人信息（姓名、年龄、性别、电话、地址）。 
    typedef struct PeoInfo {
    char name[MAX_NAME];
    int age;
    char sex[MAX_SEX];
    char tele[MAX_TELE];
    char adder[MAX_ADDER];
} PeoInfo;
  • 数组操作：使用结构体数组 struct Contact 存储联系人信息。
  • 排序算法：使用 qsort 实现按姓名和年龄排序。 
    int cmp_by_name(const void* e1, const void* e2) {
    return strcmp(((PeoInfo*)e1)->name, ((PeoInfo*)e2)->name);
}

2. 数据管理逻辑

  • 数据存储：使用结构体数组存储联系人信息，支持动态添加和删除。
  • 查找逻辑：通过姓名查找联系人记录，使用线性搜索。 
    static int findByName(const Contact* pc, char name[]) {
    for (int i = 0; i < pc->sz; i++) {
        if (strcmp(pc->data[i].name, name) == 0) {
            return i;
        }
    }
    return -1;
}
  • 删除逻辑：通过后移元素实现删除。

3. 输入输出处理

  • 格式化显示：设计了表格形式的输出，使用格式化字符串对齐。 
    void ShowContact(const Contact* pc) {
    if (pc->sz == 0) {
        printf("通讯录为空\n");
        return;
    }
    printf("%-20s %-4s %-5s %-12s %-30s\n", "姓名", "年龄", "性别", "电话", "地址");
    for (int i = 0; i < pc->sz; i++) {
        printf("%-20s %-4d %-5s %-12s %-30s\n", pc->data[i].name, pc->data[i].age,
               pc->data[i].sex, pc->data[i].tele, pc->data[i].adder);
    }
}
  • 输入验证：检查年龄、性别、电话的合法性。

4. 程序控制流

  • 循环与分支：使用 do-while 和 switch 实现菜单循环。
  • 用户交互：提供清晰的菜单选项和提示信息。

掌握的技能

1. C 编程能力

  • 熟练操作结构体数组，管理动态数据。
  • 掌握了 qsort 排序函数的使用。
  • 理解了数组的增删改查操作。

2. 数据管理实现

  • 实现了基本的 CRUD 操作（增删改查）。
  • 处理了数据删除时的数组元素移动。

3. 调试与测试

  • 通过多次测试验证了添加、删除、查找等功能的正确性。
  • 测试了边界情况（如通讯录为空、姓名重复）。

4. 用户体验设计

  • 设计了直观的表格显示和菜单交互。
  • 添加了提示信息（如“要查找的人不存在”）提升交互性。

成长与反思

1. 从基础到应用

通过这个项目，我从简单的 C 语言语法学习，走向了实际的数据管理应用，体会到了结构化编程的魅力。

2. 输入处理的挑战

在添加联系人时，我曾遇到输入缓冲区问题，通过使用 fgets 和 getchar() 解决了问题。

3. 数据管理的思考

当前系统使用数组存储数据，容量固定为 100，未来可以考虑使用链表或文件存储来支持更多联系人。

项目成果

  • 功能完整：实现了通讯录管理的核心功能，包括增删改查和排序。
  • 代码质量：代码结构清晰，注释完善。
  • 测试验证：通过多次操作验证了功能的正确性。

下一步计划

  • 动态存储：使用链表或动态数组支持更多联系人。
  • 文件存储：实现数据的持久化存储，程序退出后数据不丢失。
  • 搜索优化：使用哈希表或二叉搜索树优化查找效率。
  • 界面优化：尝试使用图形界面（如 Qt）替代命令行。

结语

通讯录管理系统是我学习 C 语言的一次重要实践。它让我掌握了结构体、数组操作和数据管理的技能，为后续学习更复杂的数据结构打下了基础。未来，我希望将这个系统集成到 Online Judge 平台中，提供更丰富的功能。
