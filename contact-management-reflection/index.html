通讯录管理系统总结 —— C 语言与数组的实践

作者: dly
完成日期: 2025-04-06
分类专栏: C 语言学习笔记
标签: C 语言, 通讯录, 数组, 排序, 文件操作
版权: 本文原创，转载请注明出处

专栏导航

本系列专栏 “C 语言学习笔记” 记录了我学习和实践 C 语言的过程，欢迎订阅以查看更多内容。

前言

我使用 C 语言实现了一个通讯录管理系统，基于静态数组管理联系人信息，支持添加、删除、查找、修改、显示和排序功能。这个项目让我深入理解了数组作为数据结构的应用，掌握了 C 语言的输入输出、指针操作和标准库函数的使用。以下是我的学习与成长总结。

项目概述

该系统是一个基于命令行的通讯录管理工具，使用静态数组存储最多 100 个联系人的信息。每个联系人包含姓名、年龄、性别、电话和地址。系统通过菜单交互实现以下功能： 1. 添加联系人：在通讯录中新增联系人。 2. 删除联系人：根据姓名删除指定联系人。 3. 查找联系人：根据姓名查找并显示信息。 4. 修改联系人：修改指定联系人的信息。 5. 显示联系人：列出所有联系人信息。 6. 排序联系人：按姓名或年龄排序。

代码分为头文件（contact.h）、实现文件（contact.c）和测试文件（test.c），总计约 250 行。

学到的知识

1. C 语言核心特性

  • 结构体定义：使用 struct PeoInfo 定义联系人信息，嵌套在 struct Contact 中管理数组。 
    typedef struct PeoInfo {
    char name[MAX_NAME];
    int age;
    char sex[MAX_SEX];
    char tele[MAX_TELE];
    char adder[MAX_ADDER];
} PeoInfo;

typedef struct Contact {
    PeoInfo data[MAX];
    int sz;
} Contact;
  • 指针与传参：通过指针（如 Contact* pc）传递结构体，确保函数能修改原始数据。
  • 枚举类型：使用 enum Option 定义菜单选项，提高代码可读性。

2. 数组管理

  • 静态数组：使用固定大小的数组（data[MAX]）存储联系人，理解了数组的容量限制。
  • 边界检查：在添加和删除时检查 sz 是否超出 MAX，避免溢出。
  • 数据移动：删除联系人时通过循环移动数组元素，保持数据连续性。

3. 输入输出处理

  • 格式化输出：使用 printf 的宽度控制（如 %-10s）美化显示效果。 
    printf("%-10s %-4s %-5s %-12s %-30s\n", "姓名", "年龄", "性别", "电话", "地址");
  • 输入优化：使用 fgets 替代 scanf 处理字符串输入，并移除换行符。

4. 标准库函数

  • 字符串操作：使用 strcmp 比较姓名，strcspn 移除换行符。
  • 排序：利用 qsort 实现按姓名和年龄排序，定义了比较函数： 
    int cmp_by_name(const void* e1, const void* e2) {
    return strcmp(((PeoInfo*)e1)->name, ((PeoInfo*)e2)->name);
}

掌握的技能

1. C 编程能力

  • 熟练操作结构体数组，例如添加联系人： 
    void AddContact(Contact* pc) {
    if (pc->sz == MAX) {
        printf("通讯录已满，无法增加\n");
        return;
    }
    printf("请输入名字：");
    fgets(pc->data[pc->sz].name, MAX_NAME, stdin);
    pc->data[pc->sz].name[strcspn(pc->data[pc->sz].name, "\n")] = 0;
    // ... 其他字段输入
    pc->sz++;
    printf("添加成功\n");
}
  • 掌握了指针和数组的协同使用。

2. 程序设计

  • 模块化：将功能拆分为独立函数（如 AddContact、DelContact），提高了代码复用性。
  • 菜单交互：使用 switch 和枚举实现用户选择逻辑。
  • 错误处理：对空通讯录、无效输入等情况进行了提示。

3. 调试与优化

  • 通过打印中间结果（如查找位置 pos）定位问题。
  • 优化输入，使用 getchar() 清除缓冲区残留换行符。

4. 算法应用

  • 实现线性查找（findByName）定位联系人。
  • 使用 qsort 实现高效排序，理解了比较函数的编写。

成长与反思

1. 从简单到复杂

最初，我只熟悉基本的 C 语法，通过这个项目，我学会了将多个功能整合成一个完整的系统，体会到了程序设计的整体性。

2. 输入处理的挑战

使用 scanf 时遇到缓冲区问题，导致输入混乱。改为 fgets 并处理换行符后，问题解决，这让我认识到输入处理的复杂性。

3. 效率意识

最初的删除操作直接清空数据，后来改为移动元素保持连续性，提升了对数组操作效率的关注。

项目成果

  • 功能完整：实现了所有要求的 6 个功能，用户可通过菜单交互管理联系人。
  • 代码质量：约 250 行 C 代码，结构清晰，包含必要的错误处理。
  • 测试验证：通过手动输入和排序验证了系统的正确性。

下一步计划

  • 动态内存：将静态数组改为动态分配，支持更多联系人。
  • 文件存储：增加数据保存和加载功能，实现持久化。
  • 多条件排序：支持按多个字段（如姓名+年龄）排序。
  • 输入验证：增加电话号码格式检查等功能。

结语

这个通讯录管理系统是我学习 C 语言的一次重要实践。它让我从基础语法走向了实用程序开发，增强了我的编程信心。未来，我希望将这些经验应用到更复杂的项目中，继续提升技能。
