顺序表实现总结 —— C 语言与动态数组的实践

作者: dly
完成日期: 2025-04-06
分类专栏: C 语言学习笔记
标签: C 语言, 顺序表, 动态数组, 数据结构, 内存管理
版权: 本文原创，转载请注明出处

专栏导航

本系列专栏 “C 语言学习笔记” 记录了我学习和实践 C 语言的过程，欢迎订阅以查看更多内容。

前言

我使用 C 语言实现了一个动态顺序表（SeqList），基于数组支持元素的尾插、头插、尾删、头删等操作，并通过测试验证了功能。这个项目让我深入理解了动态数组的实现原理，掌握了 C 语言的内存管理、指针操作和数据结构的基本操作。以下是我的学习与成长总结。

项目概述

该顺序表是一个动态数组实现的线性表，支持以下功能： 1. 初始化：创建空的顺序表。 2. 尾插：在表尾添加元素。 3. 头插：在表头添加元素。 4. 尾删：删除表尾元素。 5. 头删：删除表头元素。 6. 打印：显示表中所有元素。 7. 销毁：释放内存并重置表。

代码分为头文件（SeqList.h）、实现文件（SeqList.c）和测试文件（test.c），总计约 150 行。核心数据结构是 SL，包含动态数组指针、大小和容量。

typedef struct Seqlist {
    SQDataType* a;
    int size;
    int capacity;
} SL;

学到的知识

1. C 语言核心特性

  • 动态内存管理：使用 malloc 和 realloc 分配和扩展内存，使用 free 释放内存。 
    SQDataType* tmp = (SQDataType*)realloc(ps->a, newcapacity * sizeof(SQDataType));
  • 指针操作：通过指针（如 ps->a）访问和修改数组内容。
  • 结构体：使用 struct Seqlist 封装数组及其状态。

2. 顺序表原理

  • 动态扩容：当 size 达到 capacity 时，按 2 倍扩容（初始容量为 4），避免频繁分配内存。
  • 时间复杂度：
    • 尾插/尾删：O(1)（均摊复杂度）。
    • 头插/头删：O(N)，需要移动元素。
  • 空间管理：通过 size 跟踪有效元素数，capacity 表示总容量。

3. 内存管理细节

  • 增容策略：从 0 到 4，再按 2 倍增长，理解了空间浪费与性能的权衡。
  • 错误处理：检查 realloc 返回值，防止内存分配失败。 
    if (tmp == NULL) {
    perror("realloc fail");
    return;
}

4. 数据结构优缺点

  • 优点：随机访问效率高（O(1)）。
  • 缺点：头插和中间插入效率低（O(N)），空间可能浪费（如只用 10 个元素却分配了 100 个）。

掌握的技能

1. C 编程能力

  • 实现动态数组操作，例如尾插： 
    void SeqListPushBack(SL* ps, SQDataType x) {
    if (ps->size == ps->capacity) {
        int newcapacity = ps->capacity == 0 ? 4 : ps->capacity * 2;
        SQDataType* tmp = (SQDataType*)realloc(ps->a, newcapacity * sizeof(SQDataType));
        if (tmp == NULL) {
            perror("realloc fail");
            return;
        }
        ps->a = tmp;
        ps->capacity = newcapacity;
    }
    ps->a[ps->size] = x;
    ps->size++;
}
  • 掌握了指针和数组的协同使用。

2. 数据结构实现

  • 实现了顺序表的基本操作，理解了数组的动态扩展和元素移动。
  • 通过循环移动元素实现头插和头删： 
    void SeqListPushFront(SL* ps, SQDataType x) {
    for (int i = ps->size; i > 0; i--) {
        ps->a[i] = ps->a[i - 1];
    }
    ps->a[0] = x;
    ps->size++;
}

3. 测试与调试

  • 编写测试函数 TestSeqList1()，验证各操作的正确性。
  • 通过打印中间状态（如 SeqListPrint）定位问题。

4. 程序设计

  • 模块化：将功能拆分为独立函数（如 SeqListPushBack、SeqListPopFront），提高了代码可维护性。
  • 安全性：在删除操作中检查空表情况，避免非法访问。

成长与反思

1. 从静态到动态

最初，我只熟悉静态数组，通过实现动态顺序表，我学会了如何根据需求调整内存大小，体会到了灵活性的重要性。

2. 内存管理的挑战

在测试扩容时，曾因未正确更新 ps->a 导致内存泄漏。通过检查 realloc 的返回值并调试，我掌握了内存管理的细节。

3. 时间复杂度的认识

头插和头删需要移动所有元素，这让我意识到顺序表的局限性，并开始思考链表的优势。

项目成果

  • 功能完整：实现了初始化、插入、删除、打印和销毁功能。
  • 代码质量：约 150 行 C 代码，结构清晰，包含错误处理。
  • 测试验证：通过测试用例（如插入 1、2、3，删除头尾）验证了正确性。

下一步计划

  • 查找与修改：增加按值查找和修改元素的功能。
  • 缩容机制：当 size 远小于 capacity 时，释放多余空间。
  • 链表对比：实现单链表，比较其与顺序表在插入删除上的效率。
  • 异常处理：完善空表和内存分配失败的提示。

结语

这个顺序表项目是我学习 C 语言和数据结构的一次重要实践。它让我从静态数组管理走向动态内存分配，增强了对底层操作的理解。未来，我希望进一步探索链表和其他数据结构，提升我的编程能力。
