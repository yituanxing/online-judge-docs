前端技术选型

技术栈

  • 运行时环境: Node.js
  • 优点: 支持前端构建和包管理，跨平台。
  • 缺点: 单线程模型，CPU 密集任务较弱。
  • 原因: 提供开发和构建环境，与 NPM 无缝集成。
  • 包管理: NPM
  • 优点: 生态庞大，依赖管理方便。
  • 缺点: 包体积可能膨胀，安装速度较慢。
  • 原因: 管理前端依赖，安装 Vue3 等库。
  • 框架: Vue3
  • 优点: 响应式编程，组件化开发，性能优化。
  • 缺点: 学习曲线较陡，社区相对 React 稍小。
  • 原因: 开发效率高，生态丰富，适合现代化前端项目。
  • 路由: Vue Router
  • 优点: 与 Vue3 深度集成，支持动态路由。
  • 缺点: 配置复杂路由时稍显繁琐。
  • 原因: 实现页面导航，提供流畅的用户体验。
  • UI 库: Element Plus
  • 优点: 与 Vue3 兼容，组件丰富，开箱即用。
  • 缺点: 定制化样式需额外配置。
  • 原因: 提供一致的 UI 组件，加速开发。
  • HTTP 请求: Axios
  • 优点: 支持 Promise，拦截器功能强大。
  • 缺点: 需要手动处理错误，体积稍大。
  • 原因: 简化与后端 API 的交互。
  • Cookie 管理: Js-cookie
  • 优点: 轻量级，操作 Cookie 简单。
  • 缺点: 功能单一，仅限 Cookie 管理。
  • 原因: 管理前端用户状态（如 Token 存储）。
  • 开发环境:
  • IDE: VS Code
    • 原因: 轻量，支持 Vue 插件，调试方便。
  • 构建工具: Vite
    • 优点: 构建速度快，支持热模块替换（HMR）。
    • 缺点: 生态较新，部分插件支持不如 Webpack。
    • 原因: 提升开发效率，优化构建体验。
  • 版本控制: Git
    • 原因: 协作开发，版本管理。

选型原因

  • 现代化开发:
  • Vue3 + Vite + Node.js 代表前端最新技术趋势，保障项目前瞻性。
  • 开发效率:
  • Element Plus 和 Axios 提供开箱即用的组件和网络请求支持，NPM 简化依赖管理。
  • 用户体验:
  • Vue Router 确保页面导航流畅，Js-cookie 管理用户状态，优化交互体验。
  • 生态支持:
  • Vue3 生态（Vue Router、Element Plus）高度兼容，减少集成成本。

注意事项

  • 状态管理:
  • 当前未使用 Pinia，若需要复杂状态管理，可考虑引入。
  • 性能优化：
  • 使用 Vite 的按需加载和 Tree Shaking 减少打包体积。

相关知识

  • 组件 - Element Plus、Axios、Js-cookie 的配置。
  • 网络 - Axios 的 HTTP 请求原理。
