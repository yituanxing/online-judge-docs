后端开发

开发环境

  • 操作系统: Windows 10 专业版
    • 作用: 本地开发和测试环境，支持 Docker Desktop 和 IntelliJ IDEA。
  • IDE: IntelliJ IDEA 2023.2.6
    • 配置: 配置 JDK 17，导入 Maven 项目，启用 Spring Boot、Maven 和 Docker 插件。
    • 作用: 提供代码编辑、调试和微服务管理功能。
  • JDK: JDK 17
    • 路径: 配置在系统环境变量 JAVA_HOME，确保 Maven 和 IDEA 使用一致版本。
    • 作用: 支持 Java 17 新特性，适配 Spring Boot 3。
  • 构建工具: Maven
    • 作用: 管理项目依赖，构建微服务模块（如 oj-gateway、oj-system 等）。
  • 数据库管理工具: DBeaver
    • 作用: 管理和可视化 MySQL 数据库 xxdoj_dev，用于表结构设计和数据调试。
    • 配置: 连接本地 MySQL（deploy/dev/mysql），导入初始脚本。
  • API 测试工具: Apifox
    • 作用: 测试和调试微服务接口（如 /api/questions），生成 API 文档。
    • 配置: 导入 oj-api 的接口定义，模拟请求和验证响应。
  • API 文档工具: Swagger
    • 作用: 自动生成并展示微服务 API 文档，方便开发和联调。
    • 配置: 在 Spring Boot 中集成 Swagger，访问路径 /swagger-ui.html。
  • 项目目录:
    • oj-api: 定义 API 接口和 DTO/VO 数据结构。
    • oj-common: 通用模块，包括核心工具（oj-common-core）、Redis（oj-common-redis）、RabbitMQ（oj-common-rabbitmq）等。
    • oj-gateway: Spring Cloud Gateway 网关服务。
    • oj-modules: 业务模块：
      • oj-friend: 题目和用户服务。
      • oj-job: 定时任务服务。
      • oj-judge: 判题服务。
      • oj-system: 系统管理服务。
    • deploy/dev: 本地开发部署目录，包含 Elasticsearch、MySQL、Nacos 等配置文件。
    • user-code: 用户提交代码存储目录。
    • user-code-pool: 判题沙箱代码池（如 oj-sandbox-jdk-0）。

开发流程

  1. 环境搭建:

    • 配置 IntelliJ IDEA，导入根目录下 pom.xml，自动加载子模块。
    • 使用 DBeaver 安装并配置 MySQL，创建数据库 xxdoj_dev，初始化表（如 tb_question、tb_user）。
    • 配置 Nacos（deploy/dev/nacos/conf），启动服务并注册微服务。
    • 配置 Redis（deploy/dev/redis），用于缓存题目和会话数据。
    • 安装 Docker Desktop，运行 Elasticsearch（deploy/dev/elasticsearch）和 RabbitMQ（deploy/dev/rabbit）。
    • 配置 Swagger，启动服务后访问 /swagger-ui.html 查看 API 文档。
    • 使用 Apifox 导入 API 定义，测试接口可用性。

  2. 微服务开发:

    • 题目服务 (oj-friend):
      • 实现题目增删改查，API 路径 /api/questions，数据存储至 MySQL，热点数据缓存到 Redis。
      • 使用 Apifox 测试接口，Swagger 生成文档。
    • 判题服务 (oj-judge):
      • 使用 Docker 沙箱（user-code-pool/oj-sandbox-jdk-*）运行代码，通过 RabbitMQ 异步处理任务。
      • 验证结果与预期输出一致，记录到 MySQL。
    • 用户服务 (oj-friend):
      • 实现注册登录功能，集成阿里云短信服务（oj-common-message），会话存储至 Redis。
      • 使用 DBeaver 检查用户信息表，Apifox 测试登录接口。
    • 搜索服务 (oj-friend):
      • 配置 Elasticsearch（带 IK 分词器），实现题目全文搜索。
      • 使用 Apifox 测试搜索接口 /api/search。
    • 网关服务 (oj-gateway):
      • 配置 Spring Cloud Gateway，路由请求到各微服务。
      • 通过 Swagger 查看网关转发后的接口。

  3. 定时任务:

    • 使用 XXL-JOB（oj-job），实现过期数据清理，结合 Redis TTL 机制。
    • 在 IntelliJ IDEA 中调试定时任务逻辑。

  4. 日志管理:

    • 配置 SLF4J + Logback（oj-common-core），日志输出到 logs/。
    • 集成 ELK（Elasticsearch + Kibana），收集微服务日志（deploy/dev/elasticsearch）。
    • 使用 DBeaver 检查日志相关的数据库记录。

  5. 测试:

    • 使用 JUnit 测试各服务接口，测试代码位于 src/test/java。
    • 使用 Apifox 模拟 100 个并发提交，测试接口性能。
    • 使用 JMeter 验证系统并发能力，记录响应时间和吞吐量。

判题流程图

以下是用户提交代码到获取结果的流程：

flowchart TD
    A[用户提交代码] --> B[前端发送请求 /api/submit]
    B --> C[Spring Cloud Gateway 路由到判题服务]
    C --> D[判题服务存入 MySQL xxdoj_dev]
    D --> E[推送任务到 RabbitMQ]
    E --> F[Docker 沙箱运行代码]
    F --> G[对比输出与预期结果]
    G --> H[结果存入 MySQL]
    H --> I[Redis 更新缓存]
    I --> J[返回结果给前端]
    J --> K[用户查看结果]
