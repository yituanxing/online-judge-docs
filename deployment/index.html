部署

部署流程

  1. 后端:

    • 使用 Maven 打包微服务为 JAR 文件（mvn clean package）。
    • 构建 Docker 镜像（基于 Dockerfile），推送至镜像仓库或本地使用。
    • 使用 Docker Compose（deploy/dev/docker-compose.yml）部署微服务及依赖组件（MySQL、Nacos、RabbitMQ 等）。
    • 配置 Nginx 作为反向代理和负载均衡，路由请求至 oj-gateway。

  2. 前端:

    • 使用 Vite 构建 Vue3 项目（npm run build），生成静态文件至 dist/。
    • 将 dist/ 目录部署至 Nginx 的静态资源目录（deploy/dev/nginx/html）。
    • 配置 Nginx 重定向，确保路由正确。

  3. 基础设施:

    • 配置日志系统：
      • 使用 SLF4J + Logback，日志输出至 logs/ 目录。
      • 各微服务（如 oj-gateway、oj-friend）生成独立日志文件。
    • 配置 XXL-JOB 定时任务（deploy/dev/xxl_job），调度清理任务。
    • 配置阿里云 ECS 生产环境，包括安全组、防火墙和 SSH 访问。

  4. 文档:

    • 使用 MkDocs 构建文档（mkdocs build），生成静态站点。
    • 部署至 GitHub Pages（mkdocs gh-deploy），托管于 GitHub 仓库。

部署环境

  • 开发环境: Windows 10 专业版
    • 工具: Docker Desktop、IntelliJ IDEA、VS Code。
    • 用途: 本地开发、测试和镜像构建。
  • 生产环境: 阿里云 ECS (Linux)
    • 系统: Ubuntu 20.04 或 CentOS 8。
    • 用途: 运行微服务、前端静态资源和基础设施。

测试的部署方案

以下是测试过的多种部署方案及其特点：

方案 1：单服务器 + 备用服务器

  • 服务器 1（主服务器）:
    • 组件: Nginx、Nacos、MySQL、Redis、Elasticsearch、RabbitMQ。
    • 服务: oj-gateway（网关服务）、oj-system（系统管理）、oj-friend（题目与用户服务）、oj-judge（判题服务）、oj-job（定时任务服务）。
    • 配置: 所有组件和服务运行于同一台服务器，使用 Docker Compose 管理。
  • 服务器 2（备用服务器）:
    • 服务: oj-gateway、oj-system、oj-friend、oj-judge、oj-job。
    • 配置: 无基础设施组件，仅运行微服务，作为备用或负载分担。
  • 优点: 主服务器集中部署，易于管理；备用服务器可快速切换。
  • 缺点: 单点故障风险较高，服务器 1 负载较重。

方案 2：Windows 10 本地 + FRP 内网穿透 + 云服务器

  • 本地电脑: Windows 10
    • 组件: Nginx、Nacos、MySQL、Redis、Elasticsearch、RabbitMQ。
    • 服务: 所有微服务（oj-gateway、oj-system 等）。
    • 工具: Docker Desktop 运行容器。
  • 云服务器: 2 核 2G 3M 带宽
    • 作用: 使用 FRP（Fast Reverse Proxy）进行内网穿透，将本地服务映射至公网。
  • 配置:
    • 本地运行 FRP 客户端，云服务器运行 FRP 服务端。
    • Nginx 配置反向代理至本地服务。
  • 优点: 利用本地资源，成本低，适合开发测试。
  • 缺点: 本地网络不稳定可能影响访问，云服务器带宽受限。

方案 3：Windows 10 + 双服务器 + OpenVPN

  • 本地电脑: Windows 10
    • 作用: 开发和调试环境，运行部分服务或前端。
  • 服务器 1: 2 核 2G 3M 带宽
    • 组件: Nginx、Nacos、Redis。
    • 服务: oj-gateway、oj-system。
  • 服务器 2: 4 核 4G 1M 带宽
    • 组件: MySQL、Elasticsearch、RabbitMQ。
    • 服务: oj-friend、oj-judge、oj-job。
  • 配置:
    • 使用 OpenVPN 构建虚拟私有网络，连接本地和两台服务器。
    • Docker Compose 分部署组件和服务。
  • 优点: 分担负载，灵活性高，OpenVPN 提供安全通信。
  • 缺点: 配置复杂，网络延迟可能影响性能。

注意事项

  • 端口管理: 确保 Docker 容器端口映射正确（如 MySQL:3306、Nacos:8848）。
  • 资源监控: 云服务器需监控 CPU 和内存使用，避免超载。
  • 日志清理: SLF4J + Logback 生成的日志文件（logs/）需配置轮转策略，避免磁盘溢出。
    • 示例配置：logback-spring.xml 中设置 maxHistory 和 totalSizeCap。
  • 安全性: 配置阿里云 ECS 安全组，仅开放必要端口（如 80、443）。
