井字棋游戏总结 —— C 语言与游戏开发的实践

本系列专栏 “C 语言学习笔记” 记录了我学习和实践 C 语言的过程，欢迎订阅以查看更多内容。

前言

我使用 C 语言实现了一个简单的井字棋（Tic-Tac-Toe）游戏，支持玩家与电脑对战，包含棋盘初始化、显示、下棋和胜负判断功能。这个项目让我掌握了二维数组的使用、游戏逻辑的设计以及 C 语言的基本输入输出操作。以下是我的学习与成长总结。

项目概述

该井字棋游戏是一个基于命令行的两人对战游戏（玩家 vs 电脑），使用 3x3 的棋盘实现。玩家用 * 表示，电脑用 # 表示。游戏提供以下功能： 1. 初始化棋盘：将棋盘清空为空格。 2. 显示棋盘：打印当前棋盘状态。 3. 玩家下棋：输入坐标落子。 4. 电脑下棋：随机选择空位落子。 5. 胜负判断：检查是否有赢家或平局。

代码分为头文件（game.h）、实现文件（game.c）和主文件（test.c），总计约 200 行。核心数据结构是 3x3 的字符数组 char Board[ROW][COL]。

学到的知识

1. C 语言核心特性

  • 二维数组：使用 char Board[ROW][COL] 表示棋盘，理解了多维数组的内存布局。
  • 随机数生成：通过 srand 和 rand 实现电脑的随机落子。 
    srand((unsigned int)time(NULL));
x = rand() % ROW;
  • 函数模块化：将功能拆分为独立函数（如 InitBoard、DisPlayBoard），提高了代码可读性。

2. 游戏逻辑设计

  • 棋盘表示：用空格（' '）表示空位，* 表示玩家，# 表示电脑。
  • 胜负条件：检查行、列和对角线是否三子连线，或棋盘是否已满。 
    char IsWin(char Board[ROW][COL], int row, int col) {
    for (int i = 0; i < row; i++) {
        if (Board[i][0] == Board[i][1] && Board[i][1] == Board[i][2] && Board[i][0] != ' ')
            return Board[i][0];
    }
    // ... 检查列和对角线
    if (IsFull(Board, row, col)) return 'Q';
    return 'C';
}
  • 状态返回：用字符表示游戏状态（* 玩家胜，# 电脑胜，Q 平局，C 继续）。

3. 输入输出处理

  • 格式化显示：设计了棋盘的打印格式，使用 | 和 --- 分隔格子。 
    void DisPlayBoard(char Board[ROW][COL], int row, int col) {
    for (int i = 0; i < row; i++) {
        for (int j = 0; j < col; j++) {
            printf(" %c ", Board[i][j]);
            if (j < col - 1) printf("|");
        }
        printf("\n");
        if (i < row - 1) {
            for (int j = 0; j < col; j++) {
                printf("---");
                if (j < col - 1) printf("|");
            }
            printf("\n");
        }
    }
}
  • 输入验证：检查玩家输入的坐标是否合法和未被占用。

4. 程序控制流

  • 循环与分支：使用 do-while 和 switch 实现菜单循环，使用 while 控制游戏进程。
  • 边界检查：确保玩家输入在 1-3 范围内，并转换为数组索引（0-2）。

掌握的技能

1. C 编程能力

  • 熟练操作二维数组，例如初始化： 
    void InitBoard(char Board[ROW][COL], int row, int col) {
    for (int i = 0; i < row; i++) {
        for (int j = 0; j < col; j++) {
            Board[i][j] = ' ';
        }
    }
}
  • 掌握了指针传递数组给函数。

2. 游戏逻辑实现

  • 实现了随机电脑下棋和胜负判断逻辑。
  • 处理了玩家输入的异常情况（如坐标被占用）。

3. 调试与测试

  • 通过打印棋盘状态验证每一步下棋的正确性。
  • 测试了所有胜负情况（行、列、对角线和平局）。

4. 用户体验设计

  • 设计了直观的棋盘显示和简单的菜单交互。
  • 添加了提示信息（如“坐标被占用”）提升交互性。

成长与反思

1. 从算法到应用

最初，我只熟悉基本的循环和条件语句，通过这个项目，我学会了将逻辑转化为游戏规则，体会到了编程的趣味性。

2. 输入处理的挑战

玩家输入坐标时，我曾忘记减 1 转换为数组索引，导致越界。通过调试和调整，我掌握了用户输入与程序逻辑的协调。

3. 简洁与效率

电脑下棋使用简单随机算法，虽然功能正确，但缺乏策略性，这让我开始思考如何优化 AI。

项目成果

  • 功能完整：实现了井字棋的完整流程，包括初始化、下棋和胜负判断。
  • 代码质量：约 200 行 C 代码，结构清晰，注释完善。
  • 测试验证：通过多次对战验证了玩家和电脑的胜负逻辑。

下一步计划

  • 智能 AI：改进 ComputerMove，实现简单的策略（如优先堵截玩家或争取胜利）。
  • 动态棋盘：支持可变大小的棋盘（如 5x5）。
  • 多人模式：增加玩家对玩家的模式。
  • 界面优化：尝试使用图形库（如 SDL）替代命令行显示。

结语

这个井字棋游戏是我学习 C 语言的一次有趣实践。它让我从基础语法走向了小型游戏开发，增强了对数组和逻辑设计的理解。未来，我希望挑战更复杂的游戏项目，进一步提升编程能力。
