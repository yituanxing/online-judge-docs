学生成绩管理系统总结（V2） —— C 语言与链表的实践

作者: dly
完成日期: 2025-04-06
分类专栏: C 语言学习笔记
标签: C 语言, 学生管理系统, 链表, 文件操作, 数据管理
版权: 本文原创，转载请注明出处

专栏导航

本系列专栏 “C 语言学习笔记” 记录了我学习和实践 C 语言的过程，欢迎订阅以查看更多内容。

前言

我使用 C 语言实现了一个基于单向链表的学生成绩管理系统，支持添加、显示、排序、删除、查找、修改学生信息，并通过文件实现数据持久化。这个项目让我深入掌握了链表操作、文件读写和界面设计的技能。以下是我的学习与成长总结。

项目概述

该系统是一个命令行工具，使用单向链表存储学生信息（学号、姓名、成绩），提供以下功能： 1. 添加学生：输入学生信息并插入链表。 2. 显示信息：格式化输出所有学生记录。 3. 按成绩排序：按成绩降序排列。 4. 删除学生：根据学号删除学生。 5. 查找学生：按学号或姓名查找。 6. 修改信息：修改指定学生的学号、姓名或成绩。 7. 文件存取：从文件读取数据并保存修改。

代码分为头文件（head.h）和实现文件（main.c 等），总计约 350 行。核心数据结构是链表节点 struct node，包含学生信息和指针。

struct student {
    char num[MAX_NUM];
    char name[MAX_NAME];
    float score;
};

struct node {
    struct student data;
    struct node* next;
};

学到的知识

1. C 语言核心特性

  • 链表实现：使用 malloc 动态创建节点，通过指针连接形成单向链表。 
    void increase() {
    struct node* p = (struct node*)malloc(sizeof(struct node));
    p->next = head->next;
    head->next = p;
}
  • 字符串操作：使用 strcmp 比较字符串，strlen 计算长度。
  • 全局变量：通过 N、N_num_max 和 N_name_max 跟踪学生人数和最大字段长度。

2. 数据结构与算法

  • 单向链表：实现了头插法添加节点和遍历删除。
  • 简单排序：使用冒泡排序思想交换节点数据（非指针交换）。 
    if (p1->data.score < p2->data.score) {
    s = p1->data;
    p1->data = p2->data;
    p2->data = s;
}

3. 文件操作

  • 读写文件：使用 fscanf 和 fprintf 实现数据的持久化存储。 
    void write() {
    FILE* fp = fopen("student2.txt", "w");
    fprintf(fp, "%d\n%d\n%d\n", N, N_num_max, N_name_max);
    while (p != NULL) {
        fprintf(fp, "%s %s %.1f\n", p->data.num, p->data.name, p->data.score);
        p = p->next;
    }
    fclose(fp);
}
  • 文件创建：若文件不存在，自动创建并初始化。

4. 界面设计

  • 格式化输出：动态调整显示宽度以适应不同长度的学号和姓名。
  • 菜单交互：通过 switch 和循环实现用户选择。

掌握的技能

1. C 编程能力

  • 熟练操作链表，例如删除节点： 
    void decrease() {
    struct node* q = head, *p = head->next;
    while (p != NULL) {
        if (!strcmp(p->data.num, str)) {
            q->next = p->next;
            free(p);
            break;
        }
        q = p;
        p = p->next;
    }
}
  • 掌握了输入输出格式化（如 %.1f 控制成绩精度）。

2. 数据管理

  • 动态跟踪字段长度（N_num_max、N_name_max），优化显示效果。
  • 实现链表的增删改查功能。

3. 文件处理

  • 将链表数据保存到文件并从文件中恢复。
  • 处理文件打开失败的情况，增强程序健壮性。

4. 调试与优化

  • 通过打印中间结果（如链表为空提示）定位问题。
  • 优化显示格式，使用分隔线和对齐提升可读性。

成长与反思

1. 从数组到链表

之前我用数组实现类似系统，这次使用链表让我体会到动态内存的灵活性，尽管增加了指针管理的复杂性。

2. 文件操作的挑战

读取文件时曾因格式不匹配导致数据混乱，调整为先写入总数和最大长度后问题解决，这让我认识到文件格式设计的重要性。

3. 效率与设计权衡

排序时直接交换数据而非节点指针，虽然简单但不够高效，这让我开始思考链表排序的优化方法。

项目成果

  • 功能完整：实现了所有要求的 7 个功能，支持数据持久化。
  • 代码质量：约 350 行 C 代码，结构清晰，包含错误处理。
  • 测试验证：通过手动输入和文件读写验证了系统的稳定性。

下一步计划

  • 高效排序：使用指针交换实现真正的链表排序（如插入排序）。
  • 数据校验：增加学号唯一性检查。
  • 界面改进：集成颜色和光标控制（代码中已注释）。
  • 异常处理：完善内存分配失败的提示。

结语

这个学生成绩管理系统是我学习 C 语言的一次深入实践。它让我从基础语法走向了链表管理和文件操作，增强了对数据结构的理解和编程信心。未来，我希望优化代码并探索更复杂的管理系统。
