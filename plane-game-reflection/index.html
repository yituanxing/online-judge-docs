飞机大战游戏总结 —— C 语言与控制台开发的实践

作者: dly
完成日期: 2025-04-06
分类专栏: C 语言学习笔记
标签: C 语言, 飞机大战, 游戏开发, 链表, Windows 控制台
版权: 本文原创，转载请注明出处

专栏导航

本系列专栏 “C 语言学习笔记” 记录了我学习和实践 C 语言的过程，欢迎订阅以查看更多内容。

前言

我使用 C 语言实现了一个基于 Windows 控制台的“飞机大战”游戏，支持玩家操控飞机发射子弹击落敌机，并提供多种模式选择。这个项目让我掌握了动态链表、控制台操作、实时更新逻辑以及游戏设计的技能。以下是我的学习与成长总结。

项目概述

“飞机大战”是一个基于命令行的射击游戏，使用 20x30 的控制台界面。玩家控制飞机（Ж）发射子弹（|）击落敌机（Ψ），包含以下功能： 1. 初始化：设置飞机、子弹和敌机的初始位置。 2. 显示界面：实时更新游戏画面和得分。 3. 玩家控制：通过 WASD 移动飞机，空格发射子弹。 4. 敌机移动：敌机随机生成并下落。 5. 子弹行为：支持普通发射、追踪和“离手”模式。 6. 模式选择： - 普通模式（难度可选）。 - 破解模式（子弹追踪敌机）。 - 离手模式（自动瞄准）。 - 复活模式（撞毁后复活）。 7. 碰撞检测：检测子弹击中敌机或飞机撞毁。

代码分为主文件和部分函数实现，总计约 400 行，使用了链表管理复活模式下的飞机。

学到的知识

1. C 语言高级特性

  • 动态内存与链表：使用 malloc 创建链表节点，支持复活模式下的飞机动态生成。 
    void AddNode() {
    pnew = (Node*)malloc(sizeof(Node));
    pnew->x = rand() % HIGH;
    pnew->y = rand() % WIDTH;
    pnew->pnext = plane->pnext;
    plane->pnext = pnew;
}
  • 指针操作：通过指针更新飞机和子弹位置。

2. Windows 控制台编程

  • 光标控制：使用 gotoxy 调整光标位置，实现实时画面更新。 
    void gotoxy(int x, int y) {
    COORD pos = {x, y};
    SetConsoleCursorPosition(GetStdHandle(STD_OUTPUT_HANDLE), pos);
}
  • 颜色设置：通过 SetConsoleTextAttribute 为不同元素设置颜色。
  • 隐藏光标：使用 SetConsoleCursorInfo 提升界面美观性。
  • 键盘输入：通过 _kbhit 和 _getch 实现非阻塞输入检测。

3. 游戏逻辑设计

  • 实时更新：将游戏分为输入相关（updateWithInput）和非输入相关（updateWitoutIput）逻辑。
  • 碰撞检测：检查子弹与敌机、飞机与敌机的重叠。
  • 模式实现：
    • 普通模式：子弹直线发射。
    • 追踪模式：子弹自动追踪敌机。
    • 离手模式：飞机自动对齐敌机。
    • 复活模式：使用链表生成新飞机。

4. 时间与随机性

  • 随机生成：使用 srand 和 rand 控制敌机位置和复活坐标。
  • 速度控制：通过计数器 speed 和 NUM 调节敌机下落速度。

掌握的技能

1. C 编程能力

  • 实现复杂逻辑，例如追踪子弹移动： 
    void bulletMove() {
    if (bullet_x > enemy_x) bullet_x--;
    else if (bullet_x < enemy_x) bullet_x++;
    if (bullet_y > enemy_y) bullet_y--;
    else if (bullet_y < enemy_y) bullet_y++;
}
  • 熟练使用结构体和指针管理游戏状态。

2. 控制台界面设计

  • 设计了动态更新的游戏界面和侧边提示。 
    void scoreandtips() {
    gotoxy(50, 8);
    color(14);
    printf("游戏得分：%d ", score);
    gotoxy(50, 10);
    printf("用W A S D 分别控制飞机的移动");
}
  • 优化显示，避免闪烁（通过 gotoxy 代替 system("cls")）。

3. 游戏开发技巧

  • 实现了多模式选择，增强了游戏可玩性。
  • 处理了边界条件（如飞机不出界、子弹超出屏幕）。

4. 调试与优化

  • 通过打印坐标和状态调试碰撞检测。
  • 添加了模式选择界面和趣味提示（如“皮一下很开心”）。

成长与反思

1. 从简单到复杂

最初，我只熟悉基本的 C 语法，通过这个项目，我学会了整合输入、显示和逻辑更新，完成了首个交互式游戏。

2. 控制台开发的挑战

画面闪烁问题让我头疼，最终通过 gotoxy 局部更新解决，这让我认识到性能优化的重要性。

3. 设计思维的提升

实现多种模式让我开始思考如何平衡难度和趣味性，例如追踪模式太强，而普通模式略显单调。

项目成果

  • 功能完整：实现了飞机移动、子弹发射、敌机生成和四种游戏模式。
  • 代码质量：约 400 行 C 代码，结构清晰，包含多种交互元素。
  • 测试验证：通过多次游戏测试验证了碰撞检测和模式切换的正确性。

下一步计划

  • 智能敌机：让敌机有简单的移动策略（如左右移动）。
  • 多子弹支持：支持同时发射多颗子弹。
  • 音效改进：完善背景音乐播放（当前注释掉）。
  • 图形化界面：尝试使用图形库（如 SDL）替代控制台。

结语

“飞机大战”是我学习 C 语言的一次精彩实践。它让我从基础编程走向了游戏开发，掌握了控制台操作和动态数据管理的技能。这个项目不仅有趣，还增强了我的信心，未来我希望挑战更复杂的游戏设计。
