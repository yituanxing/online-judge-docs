NEMU 模拟器完成总结 —— YSYX 项目学习与成长记录

作者: xxd
完成日期: 2025-04-06
分类专栏: YSYX 项目笔记
标签: NEMU, YSYX, RISC-V, 模拟器, 学习总结
版权: 本文原创，转载请注明出处

专栏导航

本系列专栏 “YSYX 项目笔记” 记录了我参与一生一芯（YSYX）项目的学习历程和实践经验，欢迎订阅以查看更多内容。

前言

在 YSYX 项目中，我完成了 NEMU（Nanos-lite Emulator）模拟器的开发，这是一款轻量级的 RISC-V 指令集模拟器。通过这个过程，我不仅深入理解了计算机体系结构和指令集的实现，还锻炼了编程能力、调试技巧和系统化思维。以下是我在完成 NEMU 项目中学到的知识和技能的总结。

学到的知识

1. RISC-V 指令集的深入理解

  • 指令格式：掌握了 RV32I 指令集的六种基本格式（R、I、S、B、U、J），包括操作码、寄存器号和立即数的编码规则。
  • 执行过程：理解了指令的生命周期，从取指（Fetch）、解码（Decode）、执行（Execute）到写回（Writeback）。
  • 特权模式：接触了 RISC-V 的特权指令（如 CSR 操作），为后续学习操作系统打下基础。

2. 计算机体系结构基础

  • CPU 模拟：通过实现 NEMU，我学会了如何在软件层面模拟硬件行为，包括程序计数器（PC）、通用寄存器和内存的协同工作。
  • 内存管理：理解了内存映射的概念，学会了如何模拟物理内存和外设（如串口）的访问。
  • 时序与状态：认识到模拟器中指令执行的顺序性和状态机的设计思想。

3. C 语言高级应用

  • 指针与内存操作：在实现内存读写时，深入理解了指针操作和手动内存管理的细节。
  • 宏与预处理器：使用宏定义指令编码和调试功能，提高了代码的可读性和可维护性。
  • 动态数据结构：通过链表或数组实现了断点管理和指令历史记录，提升了对数据结构的运用能力。

4. 调试与验证方法

  • 差分测试：学会了使用 Spike（RISC-V 官方模拟器）作为参考，与 NEMU 的执行结果进行对比，验证实现的正确性。
  • 日志分析：通过打印寄存器和内存状态，掌握了如何定位程序中的逻辑错误。

4. 调试功能实现

  • 实现help命令：打印命令的帮助信息
  • 实现运行命令：继续运行被暂停的程序
  • 退出命令：退出NEMU
  • 实现单步执行命令：让程序单步执行N条指令后暂停执行, 当N没有给出时, 缺省为1
  • info指令：打印寄存器状态 ，打印监视点信息
  • 扫描内存命令：求出表达式EXPR的值, 将结果作为起始内存地址, 以十六进制形式输出连续的N个4字节
  • 表达式求值：求出表达式EXPR的值
  • 实现设置断点命令：当表达式EXPR的值发生变化时, 暂停程序执行
  • 删除监视点：删除序号为N的监视点
  • 差分测试：学会了使用 Spike（RISC-V 官方模拟器）作为参考，与 NEMU 的执行结果进行对比，验证实现的正确性。
  • 日志分析：通过打印寄存器和内存状态，掌握了如何定位程序中的逻辑错误。

掌握的技能

1. 代码设计与模块化

  • 将 NEMU 分解为多个模块（如指令解码、执行、内存管理、调试器），提高了代码的复用性和可维护性。
  • 学会了定义清晰的接口（如函数参数和返回值），确保模块间的松耦合。

2. 调试复杂系统

  • 使用 GDB 调试 NEMU 本身，掌握了设置断点、查看调用栈和变量值的方法。
  • 实现交互式调试器，支持单步执行、断点设置和状态查看，锻炼了用户界面的设计能力。

3. 项目管理与工具使用

  • Git：熟练使用 Git 管理代码版本，提交清晰的 commit 记录，便于回溯和协作。
  • Make：编写 Makefile 自动化编译和测试流程，提高了开发效率。
  • Linux 环境：在 Linux 下完成开发，熟悉了终端命令和文件操作。

4. 测试与验证

  • 编写测试用例（如简单的 RISC-V 汇编程序），验证 NEMU 的功能完整性。
  • 通过运行 Nanos-lite 等小型操作系统，测试模拟器的稳定性。

成长与反思

1. 技术能力的提升

完成 NEMU 让我从理论走向实践，将计算机体系结构的知识转化为可运行的代码。最初，我对如何将指令翻译为 C 代码感到迷茫，但通过查阅 RISC-V 手册和调试逐步攻克难关，信心显著增强。

2. 解决问题的能力

在实现过程中，我遇到了诸如指令解码错误、内存越界等问题。通过阅读源码、打印中间结果和与社区交流，我学会了系统化地分析和解决问题。

3. 时间管理与毅力

NEMU 的开发需要较长时间，我学会了将任务拆分为小目标（如先实现基本指令，再添加调试功能），逐步推进，最终完成了整个项目。这种坚持让我体会到“积少成多”的力量。

项目成果

  • 功能实现：NEMU 支持 RV32I 基本指令集，能正确运行简单的 C 程序和 Nanos-lite。
  • 调试工具：内置交互式调试器，支持断点、单步和状态查看。

下一步计划

  • 优化性能：探索如何减少模拟器的执行开销，例如使用更高效的数据结构。
  • 扩展功能：支持 RV64I 或更多特权指令，模拟更复杂的系统。
  • 硬件实现：将 NEMU 的经验应用到硬件描述语言（如 Chisel），设计真正的 RISC-V CPU。

结语

完成 NEMU 是我在 YSYX 项目中的一次重要里程碑。它不仅让我掌握了 RISC-V 和模拟器开发的技能，还培养了我面对复杂问题的勇气和耐心。未来，我希望将这些经验运用到更广泛的领域，继续探索计算机系统的奥秘。
