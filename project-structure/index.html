项目结构

以下是 Online Judge 项目的目录结构及其功能概述，反映了项目的模块化设计和文件组织方式。

项目目录结构

根目录

  • .idea: 存储 IntelliJ IDEA 的项目配置，便于开发环境复现。
  • logs: 存放微服务运行时的日志文件，便于调试和问题排查。

部署相关

  • deploy/dev: 开发环境的配置文件和数据目录。
  • elasticsearch: Elasticsearch 服务配置及插件（如 IK 分词器）。
  • mysql: MySQL 数据目录。
  • nacos: Nacos 服务配置。
  • nginx: Nginx 配置及前端静态资源。
  • rabbit: RabbitMQ 数据目录。
  • redis: Redis 数据目录。
  • xxdoj-jar: 微服务编译后的 JAR 包存放目录。

核心模块

  • oj-api: 定义所有微服务的 API 接口和数据结构。
  • oj-common: 提供通用工具和组件，供其他模块复用。
  • oj-gateway: 网关模块，统一管理外部请求并分发至各微服务。
  • oj-modules: 核心业务模块。
  • oj-friend: 处理题目和用户相关的业务逻辑。
  • oj-job: 管理定时任务。
  • oj-judge: 执行代码判题。
  • oj-system: 管理系统配置。

用户代码相关

  • user-code: 存储用户提交的代码文件，作为判题服务的输入。
  • user-code-pool: 存放 Docker 沙箱环境的预置代码和配置。

注意事项

  • 目录清理: user-code 和 user-code-pool 需定期清理以释放磁盘空间。
  • 模块依赖: oj-modules 中的子模块依赖 oj-common 和 oj-api。
  • 日志管理: logs 目录需配置日志轮转。
