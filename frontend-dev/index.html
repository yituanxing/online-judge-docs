前端开发

开发环境

  • 编辑器: Visual Studio Code
  • 作用: 提供轻量级代码编辑、调试和插件支持。
  • 核心技术:
  • Vite + Vue3
  • TypeScript
  • Element Plus
  • 插件:
  • Vue Official (Vue - Official): 提供 Vue3 语法高亮、自动补全和调试支持。
  • Auto Close Tag: 自动闭合 HTML/XML 标签，提升编码效率。
  • Fitten Code: AI 辅助代码生成和优化，加速组件开发。
  • Open in Browser: 快速在浏览器中预览页面，便于调试。
  • ESLint: 代码规范检查，确保代码质量和一致性。
  • JavaScript Code Snippets: 提供常用 JavaScript 和 Vue 代码片段，简化开发。
  • 项目目录:
  • src/: 前端源码目录。
    • components/: 可复用组件（如题目卡片、搜索框）。
    • views/: 页面视图（如题目列表、提交页面）。
    • router/: Vue Router 路由配置。
    • store/: Pinia 状态管理。
    • assets/: 静态资源（如图片、CSS）。

开发流程

  1. 环境搭建:
  2. 安装 VS Code，配置插件（Vue Official、ESLint 等）。
  3. 使用 npm create vite@latest 初始化 Vite + Vue3 项目，启用 TypeScript。
  4. 安装 Element Plus（npm install element-plus），按需导入组件。
  5. 配置 ESLint 规则（.eslintrc.js），启用代码检查。

  6. 通过 Open in Browser 插件验证项目运行。

  7. 页面开发:

  8. 题目列表:
    • 使用 Vue Router 实现路由跳转和分页功能（/questions?page=1）。
    • 集成 Element Plus 的 Table 组件展示题目。
    • 通过 Fitten Code 快速生成列表模板。
  9. 提交页面:
    • 集成 Monaco Editor，提供代码编辑功能，支持多语言高亮。
    • 使用 JavaScript Code Snippets 快速编写表单提交逻辑。
    • 调用后端接口 /api/submit，提交代码。

  10. 搜索功能:

    • 调用后端 Elasticsearch API（/api/search），展示搜索结果。
    • 使用 Element Plus 的 Input 和 Button 组件实现搜索框。
    • 通过 Auto Close Tag 加速 HTML 结构编写。

  11. 状态管理:

  12. 使用 Pinia 管理全局状态：
    • 题目状态：存储题目列表和当前选中题目。
    • 用户状态：保存登录信息和权限。

  13. 通过 Vue Official 插件调试 Pinia Store。

  14. 测试:

  15. 使用 Jest 测试组件功能，测试用例位于 tests/ 目录。
  16. 通过 ESLint 确保代码符合规范。
  17. 使用 Open in Browser 验证页面渲染效果。

挑战与解决

  • 挑战: Monaco Editor 加载慢，影响提交页面性能。
  • 解决:
  • 按需加载 Monaco Editor，仅在提交页面初始化时加载。
  • 使用 Vite 的动态导入（import()），减少首次渲染开销。
  • 通过 Fitten Code 优化代码逻辑，减少不必要计算。
  • 挑战: 多人协作时代码风格不一致。
  • 解决:
  • 配置 ESLint 和 Prettier，提交前自动格式化代码。
  • 使用 JavaScript Code Snippets 统一常用代码模板。
