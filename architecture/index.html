系统架构设计

整体架构

Online Judge 系统采用 B/S（浏览器/服务器）架构，前端通过浏览器访问，后端基于微服务设计实现功能模块化。系统分为前端、后端和基础设施三个层次，使用 Nginx 提供负载均衡，Spring Cloud Gateway 作为 API 网关统一管理请求路由。后端各微服务独立部署在单机环境中，不采用传统集群架构，通过 Docker 容器化实现轻量级部署和高可用性。
